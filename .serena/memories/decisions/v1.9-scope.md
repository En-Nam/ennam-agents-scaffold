## Origin

R&D round (2026-07-01) on VISION/DESIGN docs proposing to turn `ennam-agents-scaffold` into a multi-agent orchestration runtime. Workflow `wf_de93563c-a3c` (8 agents) verified all Claude Code primitives (Agent View, Agent Teams, hooks, `claude --bg`, teammate frontmatter caveat) are real and shipping. Reframer resolved the June 25 mission-drift tension: **ship as a scaffold PROFILE, not a scaffold RUNTIME.** Anthropic ships the runtime; our job is to lay down the config.

## CTO answers (Danny, 2026-07-01) — foundational

1. **Python assumption in DESIGN §1**: applies to the target stack's CLAUDE templates only. The profile itself (like every other profile) is stack-agnostic.
2. **Token budget**: on end-user's Anthropic account. Must disclose the Opus-orchestrator + Sonnet-workers cost profile in `CLAUDE.md.partial.hbs` of the `agent-org` profile.
3. **`minClaudeCodeVersion` field**: add to the profile registry + preflight check in wizard. Ship in v1.9.0 as foundational infra for any profile leaning on experimental Claude Code flags.

## MUST (ships in v1.9.0)

### 1. `minClaudeCodeVersion` field + wizard preflight
- **Where**: `Profile` type in `packages/cli/src/profiles.ts`; preflight function in wizard flow.
- **Behavior**: On `resolveProfile` result, if profile has `minClaudeCodeVersion`, exec `claude --version` and compare. Mismatch → WARN (not block) with actionable message. `claude` not installed → WARN with install hint. `--no-prompts` mode → suppress interactive prompt but still WARN.
- **Acceptance**: New unit tests for the preflight comparator + integration test asserting wizard prints the warning line. All 184 existing tests still green.
- **No profile in v1.9.0 yet REQUIRES the field** — infra ships first so `agent-org` can use it later.

### 2. Feature A: `--analyze-claude` detector (issue #4a)
- Scanner over target CLAUDE.md before append, printing warnings when detected sections overlap.
- **Patterns (Ian-approved, 12)**: Task Startup Protocol, Adaptive Session Boot Protocol, Skill Loading Policy, Serena MCP Policy, Session Checkpoint Protocol, Completion Signal, Workflow, Verification, Plugin / MCP Capability Discovery, Plugin Activation Matrix, End-of-Task Sentinel, Source of Truth.
- **Acceptance**: `npx @ennamjsc/agents-scaffold --analyze-claude` on a fixture CLAUDE.md prints the correct detected sections + a recommendation. Non-destructive. Estimate ~2-3h.

### 3. `qa-automation` profile (issue #2, per Alden's spec)
- 3 skills: `qa-maestro` (mobile), `qa-playwright` (web, JSDoc BDD header convention per team pattern), `gherkin-bdd` (shared parser).
- References Maestro install from `react-native` profile — does not duplicate.
- **This profile is the trial task for the meta-spike (see #4).**
- **Acceptance**: `npx ... --profile qa-automation` produces a working scaffold; profile passes the same rubric as v1.4 profiles; smoke tests green.

### 4. Meta-spike: `agent-org` profile via in-place dogfood
- Add 3-role team (`orchestrator`, `implementer`, `reviewer`) as `.claude/agents/*.md` in THIS repo + 1 `TaskCompleted` hook appending to Serena.
- Trial: use it to build `qa-automation` (item 3) end-to-end unattended.
- **Success gate**: unattended run reaches a green PR.
- **Pass** → package the pattern as `agent-org` profile shipped in v1.9.0 with `minClaudeCodeVersion: '2.1.178'` (post-`TeamCreate/Delete` removal).
- **Fail** → checkpoint failure modes as `mem:decisions/agent-teams-viability` and route to `ennam-auto-pipeline/RD-v3-teams-based.md`. v1.9.0 still ships items 1-3.

## SHOULD (ships if MUST on track)

- Update root `README.md` with `qa-automation` and (conditionally) `agent-org` rows.
- Update `CHANGELOG.md` with v1.9.0 entry including cost-disclosure note on `agent-org`.

## COULD (parked for v1.10)

- **Feature B: `--claude-strategy=minimal`** (issue #4b). Deferred per Ian's own vote ordering. Feature A gives 80% of the value; observe real conflict reports from v1.9.0 users before defining "integration bits" precisely.

## WON'T (v1.9.0 — explicit)

- **The DESIGN doc as-written.** Reject Python rewrite of scaffold itself; reject "Phase 0 discovery run on this repo" framing.
- **Any changes to marker-block format** or existing 16 profiles beyond additive.
- **Auto-merge** on the meta-spike PR. Human gate stays.
- **Adopting Agent View / Agent Teams as the scaffold's own runtime.** We emit templates; users' Claude Code is the runtime.

## Non-verified caveats

- Claude Code v2.1.178 removed `TeamCreate`/`TeamDelete` tools and deprecated `team_name` in hook payloads. Templates in `agent-org` MUST target post-178 shape.
- The two `RD-auto-issue-pipeline` specs referenced in the DESIGN live in `ennam-auto-pipeline/`, not this repo — cross-reference during agent-org design, do not import.

## Kill criteria

If the meta-spike (item 4) can't reach a green unattended PR on the `qa-automation` trial task within one focused week, kill `agent-org` for v1.9.0 and ship v1.9.0 with items 1-3 only. Do NOT force item 4 into scope.
